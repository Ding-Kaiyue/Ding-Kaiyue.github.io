笛卡尔控制的安全设计（下篇）：四大检验机制 + 容错机制的工程实现

本篇定位
上篇讲”为什么危险”，下篇讲”怎么做对”。从理论知识 → 工程工具的完整桥接。

本篇适合：想要理解CartesianVelocityController实现细节的工程师、想自己写安全笛卡尔控制器的开发者

核心观点：

  • 安全不是靠单一算法，而是靠多维检验网
  • Fallback缩放是真正救命的工程策略
  • 坐标系统一比你想象的重要得多

1. 极速回顾：为什么需要安全链

上篇讲了四大危险源：

  • 奇异点放大 - Jacobian秩亏导致速度爆炸
  • 放大效应 - 条件数大导致小输入产生大输出
  • 方向偏移 - QP求解可能给出错方向的解
  • 坐标系退化 - 奇异点附近参考系会翻转

这些问题都指向一个设计原则：不要相信任何单一的检验，要构建多维检验网。

3. 坐标系转换：工程中最容易忽略的部分

实际实现的坐标系转换：

为什么必须这样做：

  1. 用户行为一致 - 用户说”往X轴走”，永远是全局X轴，不会因为机械臂旋转而改变
  2. 奇异点附近的稳定性 - 避免参考坐标系本身在奇异点附近翻转（上篇讲的二阶退化）
  3. 多臂协调 - 双臂同时运动时，所有计算基于同一参考系

危险场景（如果不做转换）：

用户要求：0.1 m/s 沿 Z 轴向上
机械臂姿态A：世界Z轴 = 局部Y轴
→ 转换错误，机械臂往错方向运动
机械臂姿态B：世界Z轴 = 局部-Z轴
→ 再次转换错误，方向反了

4. 四大检验机制的工程实现

4.1 前置几何可行性检验 check_direction_feasibility()

目的：在QP求解前快速判断方向是否物理可实现

核心思想：通过SVD分解，看用户要求的速度方向是否落在Jacobian的“可达方向”内

代码中的实现逻辑：

关键思想很简单：把用户要求的速度 v 拆分成两部分

  • 可以达到的部分：v_proj
  • 无法达到的部分：v - v_proj

然后计算三个比值：

比值1：可达比例 = ||v_proj|| / ||v||
如果这个比例 < 95%，说明超过5%的速度无法达到 → 停止

比值2：残差大小 = ||v - v_proj||
如果这个残差 > 10% * ||v||，说明无法达到的部分太多 → 停止

比值3：条件数 = σ_max / σ_min
如果条件数 > 1000000，说明接近奇异点 → 停止

4.2 QP求解 solve_velocity_qp()

核心问题：用户要求机械臂以0.1 m/s移动，但关节可能有速度限制（比如每个关节最多只能转10°/s）。这时候问题变成：

  • 在关节速度限制的约束下
  • 尽可能完成用户的速度命令
  • 同时避免关节速度爆炸

解决办法：引入缩放因子s

不是非要完成100%的命令，而是求解”最多能执行多少”。用一个0到1之间的数s来表示：

  • s = 1.0：完全执行，机械臂以0.1 m/s运动
  • s = 0.5：打折执行，机械臂以0.05 m/s运动
  • s = 0：无法执行，机械臂停止

QP的目标：

最大化s（让机械臂尽可能快地执行命令），同时防止关节速度过大。用数学语言：

$$\text{minimize} \quad \frac{1}{2}\varepsilon |\dot{q}|^2 + \frac{1}{2}\varepsilon_s s^2 - s$$

翻译成人话就是：

  • 减少关节速度（前一项）
  • 减少s的波动（中间项，很小，几乎不起作用）
  • 最重要的：最大化s（后面的 $-s$ 项，因为最小化 $-s$ 就是最大化s）

约束条件：

关节必须满足 Jacobian 关系： J * 关节速度 = s * 用户命令

关节速度不能超过限制： 关节速度_最小 ≤ 关节速度 ≤ 关节速度_最大

缩放因子必须有效： 0 ≤ s ≤ 1

两个神奇的数字：

为什么这样设置？

  • eps = 1e-4：这个罚款”刚好”，不会太严格也不会太松散

    • 太小（比如1e-10）：靠近奇异点时关节速度会爆炸，数值不稳定
    • 太大（比如0.01）：QP会过度抑制关节速度，即使能执行也被限制了

  • eps_s = 1e-8：故意设得远远小于eps，目的是设置”优先级”

    • 这样QP会优先最大化s（完成命令）
    • 其次才是最小化关节速度（平稳运动）
    • 换句话说：宁可关节速度大一点，也要把用户命令完成

求解失败时：如果QP求不出来，就缩小用户命令（乘以0.9），重新试。最多试6次。

4.4 关节限制检查 check_joint_pos_boundary()

问题：即使QP求解合法，关节速度也可能把某个关节推向物理限制（比如肩关节转不了这么多）。

核心思想：预测下一个控制周期（10ms后）关节会转到哪里，检查是否会超出硬限制。

三个检查场景：

当前状态 预测状态 动作 说明
在限制内 在限制内 ✓ 允许 正常运动
在限制内 超出限制 ✗ 拒绝 防止越界
已超出限制 继续超出 ✗ 拒绝 防止更坏
已超出限制 恢复回来 ✓ 允许 让机械臂脱离限制

5. Fallback缩放：真正救命的工程策略

5.1 什么是Fallback

关键原则：只有在”接近奇异点”时才用Fallback缩放，如果是”结构性无解”就直接停止。

两种失败情况：

失败原因 判定方法 处理方式
接近奇异点（可恢复） 条件数大、但方向可达 → 进入Fallback缩放
结构性奇异点（无法恢复） 方向本身不可达（feasibility < 95% 或 residual > 10%） → 直接停止

Fallback的流程（仅用于可恢复情况）：

  1. 缩小用户输入的速度（乘以0.9）
  2. 重新执行整个检验流程
  3. 如果通过，就以这个缩小的速度运动
  4. 最多重试 MAX_ITERS 次（通常6次）
  5. 如果都失败，输出零速度

为什么要区分？ 缩放速度只对”量”的问题有效，对”质”的问题无效：

  • “量”的问题：速度太大 → 缩小速度 → 解决 ✓
  • “质”的问题：方向不可达 → 缩小速度 → 还是不可达 ✗

5.2 缩放公式与速度衰减

公式：v = v * 0.9^iter

真实例子：

迭代 缩放因子 用户要求 实际执行
0 1.0 0.1000 m/s 0.1000 m/s
1 0.9 - 0.0900 m/s
2 0.81 - 0.0810 m/s
3 0.729 - 0.0729 m/s
4 0.6561 - 0.0656 m/s
5 0.59 - 0.0590 m/s
6 0.531 - 0.0531 m/s

为什么是0.9而不是0.5？

  • 0.5：降速太快，通常2次缩放就接近0，效率低
  • 0.9：下降温和，通常3-5次缩放能从”接近奇异点”恢复到”安全区”

5.3 为什么Fallback比伪逆好

什么是伪逆？

简单说，就是用一个公式直接计算关节速度：

$$关节速度 = 伪逆矩阵 × 用户命令$$

这个公式在大多数情况下很快，不需要QP求解。

伪逆的致命问题：

接近奇异点时，伪逆矩阵的某些数字会变得极其巨大（数学上叫”爆炸”）。

比如：

  • 一秒前：0.1 m/s 的命令 → 关节速度 10°/s
  • 现在靠近奇异点：0.1 m/s 的命令 → 关节速度 1000°/s！
  • 下一秒又远离奇异点：0.1 m/s 的命令 → 关节速度 5°/s

用户体验：机械臂会突然猛速转动，然后又变慢，看起来像在抖动。而且这种跳变会导致电机超负荷。

Fallback的优势：

场景 伪逆 Fallback
奇异点附近 ⚠️ 关节速度瞬间爆炸 ✓ 缩小速度，平稳降速
恢复过程 ⚠️ 又会瞬间跳回小速度 ✓ 逐步增速，连续平滑
安全检验 ❌ 无，直接输出 ✓ 每次都检验
用户感受 “机器在抖” “机器变慢了”
